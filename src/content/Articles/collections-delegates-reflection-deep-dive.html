MODULE 01 What Are Collections? Concepts #1–#4
#01
What is a Collection?

A Collection in C# is a class that groups related objects together so you can work on them as a single unit — add, remove, find, sort.

Think of a collection as a smart array — it can grow/shrink dynamically and provides built-in operations.
ARRAY vs COLLECTION
✗ Array
Fixed size
int[] arr = new int[5];
Can't resize
✓ Collection
Dynamic size
ArrayList list = new();
Add/Remove freely
System.Collections System.Collections.Generic IEnumerable
#02
Why Use Collections?
  • Dynamic sizing — grow/shrink at runtime without declaring size
  • Built-in operations — sort, search, filter, enumerate
  • Type safety — generics prevent incorrect types
  • Rich API — LINQ works on all collections
  • Productivity — less code for common tasks
Real scenario: A shopping cart has unknown item count. Use List<CartItem> — items added/removed dynamically.
USE WHEN
Data count unknown, frequent add/remove, LINQ queries needed
AVOID WHEN
Fixed data, max performance needed, no type flexibility

MODULE 02 ArrayList & Hashtable Concepts #5–#8
#05
ArrayList — What & How

ArrayList is a non-generic, dynamically resizable array. It stores object references — any type can be stored.

⚠️ Not type-safe. Boxing/unboxing happens for value types. Prefer List<T> in modern C#.
C# using System.Collections; ArrayList list = new ArrayList(); list.Add("Alice"); // string list.Add(42); // int — boxed to object list.Add(true); // bool list.Remove("Alice"); list.RemoveAt(0); list.Insert(0, "Bob"); list.Sort(); // sorts if uniform types foreach (var item in list) Console.WriteLine(item); // Boxing/Unboxing penalty: int val = (int)list[1]; // explicit cast needed

#06
Hashtable — Key-Value Store

Hashtable stores data as key-value pairs. Keys must be unique. Both key & value are object — not type-safe.

⚠️ Non-generic. Use Dictionary<TKey,TValue> instead in modern code.
C# Hashtable ht = new Hashtable(); // Add key-value pairs ht.Add("Name", "Alice"); ht.Add("Age", 30); ht.Add("City", "Mumbai"); // Access by key string name = (string)ht["Name"]; // Check existence if (ht.ContainsKey("Age")) { ... } // Remove ht.Remove("City"); // Iterate (order NOT guaranteed) foreach (DictionaryEntry entry in ht) Console.WriteLine($"{entry.Key}: {entry.Value}");
Real world use: Config settings store — ht["ConnectionString"] before generic era. Today use Dictionary<string,string>.
#07
SortedList — Non-Generic

SortedList maintains key-value pairs sorted by key automatically. Internally uses parallel arrays for keys and values.

  • Always sorted by key (ascending)
  • O(log n) binary search for retrieval
  • Access by key or index
  • Not thread-safe by default
C# SortedList sl = new SortedList(); sl.Add(3, "Charlie"); sl.Add(1, "Alice"); sl.Add(2, "Bob"); // Automatically sorted by key // Output: 1-Alice, 2-Bob, 3-Charlie // Access by key string val = (string)sl[1]; // "Alice" // Access by index string v = (string)sl.GetByIndex(0);
MODULE 03 Stack & Queue (Non-Generic) Concepts #9–#12
#09
Stack — LIFO (Last In, First Out)

Stack = LIFO structure. Last item pushed is first item popped. Like a stack of plates.

Push(3) → Top
Push(2)
Push(1) → Bottom
Pop() → returns 3 first
C# Stack s = new Stack(); s.Push("A"); s.Push("B"); s.Push("C"); s.Pop(); // "C" (LIFO) s.Peek(); // "B" (no remove) s.Count; // 2 s.Contains("A"); // true
Real world: Browser back button, undo/redo in editors, call stack in debugging, expression evaluation (parenthesis matching).
Push()Pop()Peek()CountContains()

Real world: Print queue, request processing, message broker (simplified), task scheduler, order processing in e-commerce.

What is List<T>?

List<T> is the most commonly used generic collection. It's a type-safe, dynamically resizing array. T is the type parameter — set at compile time.

COMPLETE API OVERVIEW
C# List<string> names = new List<string>(); // Add names.Add("Alice"); names.AddRange(new[] { "Bob", "Charlie" }); names.Insert(1, "Dave"); // at index 1 // Access string first = names[0]; // O(1) int idx = names.IndexOf("Bob"); // Search bool has = names.Contains("Alice"); string found = names.Find(x => x.StartsWith("A")); List<string> all = names.FindAll(x => x.Length > 3); // Sort names.Sort(); names.Sort((a, b) => a.CompareTo(b)); // Remove names.Remove("Dave"); names.RemoveAt(0); names.RemoveAll(x => x.StartsWith("C")); // Conversion string[] arr = names.ToArray(); List<string> copy = names.ToList();

Add 5th item → Capacity = 8
Add 9th item → Capacity = 16
Always doubles!
Performance tip: If you know size upfront use new List<T>(capacity) to avoid unnecessary re-allocations.

Dictionary — Fast Key-Value Lookup

Dictionary<TKey,TValue> stores unique-key to value mappings. Uses hash table internally — O(1) average lookup.

MODULE 06 HashSet<T>, Queue<T>, Stack<T> Concepts #24–#28
#24
HashSet<T> — Unique Values Only

HashSet<T> stores only unique elements with O(1) lookup. No duplicates allowed. Unordered collection.

C# var visited = new HashSet<string>(); visited.Add("pageA"); visited.Add("pageB"); visited.Add("pageA"); // duplicate — ignored! visited.Count; // 2 (not 3) visited.Contains("pageA"); // O(1)! // Set operations var setA = new HashSet<int> { 1,2,3,4 }; var setB = new HashSet<int> { 3,4,5,6 }; setA.IntersectWith(setB); // {3,4} setA.UnionWith(setB); // {1,2,3,4,5,6} setA.ExceptWith(setB); // {1,2} setA.IsSubsetOf(setB); // false
Real world: Unique visitor tracking, removing duplicates from data, role-permission checks, graph traversal (visited nodes).

MODULE 08 Delegates — Basics Concepts #35–#40
#35
What is a Delegate?

A delegate is a type-safe function pointer — it holds a reference to a method and can invoke it later. Delegates are reference types.

Think of a delegate as a variable that stores a method the same way an int variable stores a number.
Mental Model
Delegate Variable
→ points to →
Actual Method
Same as: int x → stores → 42
C# // 1. Declare delegate type delegate void Greet(string name); // 2. Method matching signature void SayHello(string name) => Console.WriteLine($"Hello, {name}!"); void SayHi(string name) => Console.WriteLine($"Hi, {name}!"); // 3. Create delegate instance Greet greetDelegate = SayHello; // 4. Invoke — calls SayHello greetDelegate("Alice"); // Hello, Alice! // 5. Reassign greetDelegate = SayHi; greetDelegate("Bob"); // Hi, Bob!
#36
Why & When Use Delegates?
  • Callbacks — pass method as argument to another method
  • Event handling — foundation of the event system
  • Strategy pattern — swap algorithms at runtime
  • LINQ — Func delegates power all LINQ queries
  • Async callbacks — completion handlers
  • Decoupling — caller doesn't know callee
Key insight: Delegates enable behavioral parameterization — passing what TO DO as a parameter, not just what data.
WHEN TO USE
Callbacks Events Strategy Pattern LINQ/Lambda Plugin system
DELEGATE RULES
  • Method signature must match delegate declaration exactly
  • Delegates are immutable reference types (like strings)
  • Can reference instance or static methods
  • Null delegates throw NullReferenceException — use ?.
#37
🎯 Delegate as Callback — Real World Example
SCENARIO: Sort with Custom Comparison
C# // Delegate type for comparison delegate int Comparer<T>(T a, T b); void BubbleSort<T>(List<T> list, Comparer<T> compare) { for (int i = 0; i < list.Count - 1; i++) for (int j = 0; j < list.Count - i - 1; j++) if (compare(list[j], list[j+1]) > 0) (list[j], list[j+1]) = (list[j+1], list[j]); } var nums = new List<int> { 5,3,8,1,9 }; BubbleSort(nums, (a, b) => a - b); // ascending BubbleSort(nums, (a, b) => b - a); // descending
SCENARIO: Progress Reporting
C# delegate void ProgressCallback(int percent); void ProcessData(int[] data, ProgressCallback onProgress) { for (int i = 0; i < data.Length; i++) { DoWork(data[i]); int pct = (i + 1) * 100 / data.Length; onProgress?.Invoke(pct); // safe invoke } } // Console app ProcessData(bigArray, p => Console.Write($"\rProgress: {p}%")); // GUI app ProcessData(bigArray, p => progressBar.Value = p);
Power of delegates: ProcessData doesn't know HOW to report progress — the caller decides. The processing logic and the UI are completely decoupled.
MODULE 09 Func, Action & Predicate Concepts #41–#45
#41
Built-in Delegate Types: Func, Action, Predicate
Func<T, TResult>
Takes input, returns a value. Last type param is return type.
C# // No params, returns int Func<int> getNum = () => 42; // Takes string, returns int Func<string,int> len = s => s.Length; // Two params, returns bool Func<int,int,bool> isSum = (a,b) => a+b > 10; // In LINQ Select var lengths = names .Select(Func<string,int> n => n.Length);
Action<T>
Takes input, returns void. Used for side effects.
C# // No params, void Action greet = () => Console.WriteLine("Hi"); // Takes string, void Action<string> log = msg => Console.WriteLine($"[LOG] {msg}"); // Two params, void Action<string,int> addUser = (name, age) => db.Insert(name, age); // In List.ForEach names.ForEach(Action<string> n => Print(n)); greet.Invoke();
Predicate<T>
Takes input, returns bool. Specialized Func<T, bool>.
C# // Check condition on T Predicate<int> isEven = n => n % 2 == 0; Predicate<string> isLong = s => s.Length > 5; isEven(4); // true isEven(7); // false // List.FindAll uses Predicate var nums = new List<int>{1,2,3,4,5}; var evens = nums.FindAll(isEven); // {2, 4}
Rule of thumb: Use Func when you need a return value, Action for side effects (void), Predicate for bool conditions. These eliminate the need to declare custom delegate types.

Compose processing steps using Func delegates.

MODULE 10 Multicast Delegates & Lambda Expressions Concepts #46–#50
#46
Multicast Delegate — Multiple Methods in One

A delegate can hold references to multiple methods. When invoked, all methods are called in order. Use += to add, -= to remove.

📧 SendEmail(message)
📱 SendSMS(message)
🔔 PushNotification(message)
notify("Order shipped!") → calls all three
C# delegate void Notify(string msg); void SendEmail(string m) => Email.Send(m); void SendSMS(string m) => SMS.Send(m); void PushNote(string m) => Push.Send(m); Notify notify = SendEmail; notify += SendSMS; // add to chain notify += PushNote; // add to chain notify("Order shipped!"); // calls ALL THREE notify -= SendSMS; // remove from chain notify("Payment done"); // calls Email + Push only // Get invocation list var list = notify.GetInvocationList(); Console.WriteLine(list.Length); // 2
⚠️ Caution: If any method throws, subsequent methods won't run. For production, invoke each individually with try-catch.
#47
Lambda Expressions with Delegates

Lambda expressions are shorthand for writing anonymous methods inline. They're the most common way to use delegates today.

C# // Anonymous method (old style) Action<string> printOld = delegate(string s) { Console.WriteLine(s); }; // Lambda equivalent Action<string> print = s => Console.WriteLine(s); // Multi-line lambda Func<int, int> factorial = n => { if (n <= 1) return 1; return n * factorial(n - 1); }; // Capture variables (closure) int multiplier = 3; Func<int, int> triple = x => x * multiplier; triple(5); // 15 — captures 'multiplier' multiplier = 10; triple(5); // 50 — uses updated 'multiplier'!
Closures: Lambda captures variables from the enclosing scope by reference. Changing the captured variable affects the lambda's result.

"Required" : null) .AddRule(s => s?.Length < 3 ? "Too short" : null) .AddRule(s => s?.Length > 50 ? "Too long" : null); var errors = v.Validate("Hi"); // ["Too short"]
MODULE 11 Events — What, Why, How Concepts #51–#56
#51
What is an Event?

An event is a mechanism for a class to notify other classes when something happens. Events are built on delegates — they are delegate fields with controlled access.

Publisher
Button, Timer, OrderService
Raises event →
event
Subscribers
LogService.OnOrder()
EmailService.SendConfirm()
InventoryService.Update()
Events vs Delegates: An event is a delegate that can only be invoked from within the class that declares it. Subscribers can only subscribe (+=) or unsubscribe (-=) — they can't invoke it directly.
event keyword EventHandler Publisher Subscriber += / -=
#52
Basic Event Declaration & Usage
C# // Publisher class public class Button { // Declare event using EventHandler delegate public event EventHandler Clicked; // Raise the event public void Click() { Console.WriteLine("Button clicked!"); // Raise: null-check and notify every subscriber currently attached. Clicked?.Invoke(this, EventArgs.Empty); } } // Subscriber var btn = new Button(); // Subscribe btn.Clicked += (sender, e) => Console.WriteLine("Handler 1: Button was clicked!"); btn.Clicked += (sender, e) => Console.WriteLine("Handler 2: Logging the click..."); // Trigger btn.Click(); // Button clicked! // Handler 1: Button was clicked! // Handler 2: Logging the click... // Unsubscribe btn.Clicked -= /* same handler ref */;

MODULE 12 Publisher-Subscriber Pattern & Real World Events Concepts #57–#62
#57

What is Reflection?

Reflection is the ability of a program to inspect and modify its own structure and behavior at runtime. Using reflection, you can discover types, methods, properties, and attributes, and invoke them dynamically.

Analogy: Reflection is like looking in a mirror — the program can see itself and act on what it sees.

🗄️ Real World: Simple ORM (Like EF Core)
C# // Attribute to mark table/column names [Table("users")] public class User { [Column("user_id")] public int Id { get; set; } [Column("user_name")] public string Name { get; set; } [Column("user_email")] public string Email { get; set; } } // Micro ORM — builds SQL from reflection public static string BuildInsert<T>(T obj) { // Inspect the runtime type so the same method can build SQL for any model. var type = typeof(T); var table = type.GetCustomAttribute<TableAttribute>()?.Name ?? type.Name; var cols = new List<string>(); var vals = new List<string>(); foreach (var prop in type.GetProperties()) { // Read optional [Column] metadata; if missing, fall back to the property name. var colAttr = prop.GetCustomAttribute<ColumnAttribute>(); var colName = colAttr?.Name ?? prop.Name; // Pull the value out of the current object instance via reflection. var value = prop.GetValue(obj); cols.Add(colName); vals.Add($"'{value}'"); } // Stitch column names and values into one INSERT statement. return $"INSERT INTO {table} ({string.Join(",",cols)}) VALUES ({string.Join(",",vals)})"; } // Usage var user = new User { Id=1, Name="Alice", Email="alice@x.com" }; string sql = BuildInsert(user); // INSERT INTO users (user_id,user_name,user_email) VALUES ('1','Alice','alice@x.com')
This is exactly how Entity Framework Core, Dapper, and AutoMapper work internally!

#QA-01
📦 Collections & Generic Collections — Q&A
Basic
What is the difference between Array and ArrayList?
Array has fixed size and is strongly typed. ArrayList is dynamically resizable and stores object — not type-safe. ArrayList requires boxing for value types.
What is boxing and unboxing? Why is it a problem with ArrayList?
Boxing wraps a value type (e.g., int) in an object heap allocation. Unboxing extracts it. With ArrayList, every int stored/retrieved is boxed/unboxed — this causes GC pressure and performance loss.
What is IEnumerable vs ICollection vs IList?
IEnumerable = read-only forward iteration. ICollection adds Count/Add/Remove. IList adds index-based access. They form a hierarchy: IEnumerable ⊂ ICollection ⊂ IList.
How does Stack differ from Queue?
Stack = LIFO (Last In, First Out). Push adds to top, Pop removes from top. Queue = FIFO (First In, First Out). Enqueue adds to back, Dequeue removes from front.
Intermediate
What is the time complexity of List<T> Add vs Insert?
Add() is O(1) amortized — appends to end. Insert(i, item) is O(n) — all elements after index i must shift right. For frequent mid-insertions, use LinkedList<T>.
When should you use HashSet<T> over List<T>?
Use HashSet when uniqueness is required or when you need O(1) Contains(). List.Contains() is O(n). For checking "has this user already visited?", HashSet is ideal.
What happens if you add a duplicate key to Dictionary?
Calling dict.Add(key, val) throws ArgumentException. Using dict[key] = val silently overwrites. Use TryAdd() in .NET 5+ to add only if key doesn't exist.
What is the internal data structure of Dictionary<K,V>?
A hash table. Keys are hashed to bucket indices. Collisions handled by chaining or open addressing. Average O(1) ops. Worst case O(n) if many collisions (poor hash function).
Advanced
How does List<T> resize internally?
When Count == Capacity, a new array of double the size is allocated and all elements are copied. Initial capacity is 4 (or the provided value). Amortized O(1) per add because resize happens infrequently.
What is ConcurrentDictionary and when to use it?
ConcurrentDictionary<K,V> is thread-safe using fine-grained locking. Use when multiple threads read/write the dictionary. Use GetOrAdd(key, factory) and AddOrUpdate() for atomic operations.
What is IReadOnlyList vs IList? Why use read-only collections?
IReadOnlyList<T> exposes only read operations (index, Count, enumeration). Return it from APIs to prevent callers from modifying internal state. Prefer for immutability and safer APIs.
Explain IEnumerable deferred execution with a code example.
LINQ queries on IEnumerable are not executed until enumerated. Where(), Select() return lazy iterators. Calling ToList() forces immediate execution. This means the query runs once at enumeration time, not at definition time.
#QA-02
🎯 Delegates — Q&A
Basic
What is a delegate in C#?
A delegate is a type-safe function pointer — a reference type that holds a reference to a method with a specific signature. It enables passing methods as arguments and storing them in variables.
What is the difference between Func, Action, and Predicate?
Func<T,R> takes inputs and returns a value. Action<T> takes inputs, returns void. Predicate<T> takes one input, returns bool. They eliminate needing to declare custom delegate types.
What is a multicast delegate?
A delegate that holds references to multiple methods. Use += to add and -= to remove. When invoked, all methods are called in order. If one throws, subsequent methods won't run.
Intermediate
What is a closure in C#? What are the risks?
A closure is a lambda that captures variables from the enclosing scope by reference. Risk: if a captured variable changes, the lambda sees the new value. Common bug in loops: for(int i=0;...) list.Add(()=>Print(i)); — all lambdas capture same i.
Can a delegate point to multiple methods with different return types?
No. A delegate has a fixed signature (return type + parameter types). All methods in a multicast delegate must share the same signature. For multicast with non-void return, only the last method's return value is captured.
What is the difference between anonymous method and lambda?
Anonymous methods use the delegate keyword: delegate(int x) { return x*2; }. Lambdas are syntactic sugar: x => x*2. Lambdas are shorter and support expression trees. Anonymous methods support discarding params.
Advanced
What are expression trees and how do they relate to delegates?
Expression trees represent code as data structures. Expression<Func<T,bool>> stores a lambda as an AST that can be inspected. Entity Framework uses expression trees to translate C# lambdas to SQL queries.
How do you safely invoke a multicast delegate when each handler might throw?
Use GetInvocationList() to get each handler separately, then invoke each in a try-catch: foreach(var d in evt.GetInvocationList()) { try { d.DynamicInvoke(args); } catch(ex){...} }
What is Func vs delegate performance difference?
Functionally identical at runtime. Func/Action are just pre-defined generic delegate types in the BCL. No performance difference. Use built-in types to avoid declaring custom delegate types for standard signatures.
#QA-03
⚡ Events — Q&A
Basic
What is the difference between a delegate and an event?
An event is a delegate with restricted access: outsiders can only subscribe/unsubscribe (+=/-=), not invoke it or assign (=) to it. Events enforce the publisher-subscriber pattern and prevent external code from triggering them.
What is EventHandler and EventHandler<T>?
EventHandler is a built-in delegate: void (object sender, EventArgs e). EventHandler<TEventArgs> adds a generic EventArgs. Use EventHandler<MyArgs> to pass custom data with events.
Why should you use ?.Invoke() when raising an event?
If no subscribers exist, the event delegate is null. event?.Invoke() safely skips invocation when null. Without it, you'd get a NullReferenceException. It's also thread-safe for the null check.
Intermediate
What is a memory leak caused by events?
When an object subscribes to an event but never unsubscribes, the publisher holds a reference preventing GC. The subscriber lives as long as the publisher. Fix: always unsubscribe in Dispose() or use WeakReference patterns.
How do you make an event thread-safe?
Copy the delegate before null-check and invocation: var handler = MyEvent; handler?.Invoke(this, e); This prevents a race where another thread unsubscribes between the null check and invocation.
What is the Observer pattern and how does it relate to C# events?
Observer pattern = objects (observers) subscribe to a subject's state changes. C# events are the language-level implementation. Event = subject, event handlers = observers. The IObservable<T>/IObserver<T> interfaces are the formal Rx version.
Advanced
What are custom event accessors (add/remove)?
You can manually control subscribe/unsubscribe with add { } and remove { } accessors — like property getters/setters. Used for thread-safe events (lock), filtered subscriptions, or lazy initialization of handler storage.
How would you implement an async event handler?
Events don't natively support async. Options: (1) async void handlers (fire-and-forget, exceptions swallowed), (2) Use Func<Task> delegates instead, (3) Use AsyncEventHandler pattern and await Task.WhenAll() over invocation list.
When would you use events over direct method calls?
When: (1) multiple subscribers need notification, (2) publisher shouldn't know about subscribers (decoupling), (3) behavior can be extended by external code (plugins), (4) subscribers can be added/removed at runtime. Use direct calls when one-to-one communication is sufficient.
#QA-04
🔍 Reflection — Q&A
Basic
What is Reflection in C#?
Reflection is the ability to inspect and manipulate type metadata at runtime using System.Reflection namespace. You can discover types, invoke methods, read/write properties, and read attributes without knowing the type at compile time.
What are the 3 ways to get a Type object?
1. typeof(MyClass) — compile-time, no instance needed. 2. myObj.GetType() — from an existing instance. 3. Type.GetType("Namespace.ClassName") — by fully qualified string name.
What is Activator.CreateInstance()?
Activator.CreateInstance(type) creates an object of the given type at runtime without using new. Uses the parameterless constructor. Can also pass constructor arguments. This is how DI containers create services.
Intermediate
What are BindingFlags and when do you need them?
BindingFlags control what members are returned by GetMethod/GetProperty etc. Without flags, only public instance members are returned. Use BindingFlags.NonPublic | BindingFlags.Instance to access private members (useful for unit testing).
Why is Reflection slow and how do you mitigate it?
Reflection has ~10-100x overhead from metadata lookup, security checks, and boxing. Mitigate by: (1) caching MethodInfo/PropertyInfo objects, (2) using compiled expression trees, (3) generating IL via Emit, (4) using source generators (compile-time reflection).
How are custom attributes read at runtime?
Call member.GetCustomAttribute<MyAttr>() on a Type, MethodInfo, or PropertyInfo. Returns null if not present. Use GetCustomAttributes() for all attributes. IsDefined(typeof(MyAttr)) for existence check.
Advanced
What are compiled expression trees and how do they help with reflection perf?
Instead of calling MethodInfo.Invoke() each time, compile a lambda delegate once: var getter = Expression.Lambda<Func<T,P>>(...).Compile(). First call is slow (compilation), subsequent calls are as fast as direct invocation. Used by EF Core, AutoMapper.
What is the difference between early binding and late binding?
Early binding: type resolved at compile time (normal C# code, IntelliSense, type checking). Late binding: type resolved at runtime via reflection or dynamic. Late binding is flexible but slower and loses compile-time safety.
What are Source Generators and how do they replace reflection?
Source generators run at compile time to generate C# code. They do what reflection does at runtime but at compile time — zero runtime overhead, full type safety. Used by System.Text.Json serialization, EF Core model building, AutoMapper in .NET 7+.
⚡ Quick Reference
Most used collection?
List<T> for sequences, Dictionary<K,V> for lookups. 80% of cases covered by these two.
Fastest Contains() check?
HashSet<T> — O(1). Dictionary.ContainsKey() — O(1). List.Contains() — O(n). Use HashSet for membership tests.
Delegate vs Interface for callback?
Use delegate for single-method callbacks. Interface for multi-method contracts (ILogger). Delegates are lighter and more flexible for single behaviors.
When does event cause memory leak?
When short-lived object subscribes to event on long-lived object and never unsubscribes. Publisher keeps subscriber alive via event chain.
Reflection in production code?
Cache MethodInfo/PropertyInfo. Never call reflection in hot loops. Consider Expression trees or source generators for performance-critical paths.
