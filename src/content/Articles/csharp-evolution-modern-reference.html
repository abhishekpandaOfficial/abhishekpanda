C# 12 · 01Primary ConstructorsClasses & Structs
✦ New in C# 12
What Are Primary Constructors?

Primary constructors let you declare constructor parameters directly on the class/struct declaration — removing the boilerplate of a separate constructor and field assignments.

Previously only record types had primary constructors. C# 12 brings this to all classes and structs.
  • Parameters are in scope throughout the entire class body
  • Parameters are NOT auto-stored as fields — you choose what to capture
  • Works for classes, structs, record classes, record structs
  • Reduces DI boilerplate by ~50% in typical services
classstructDIServices
Before vs After
✗ C# 11 — Verbose
public class OrderService { // 3 separate steps needed: private readonly IRepo _repo; private readonly ILogger _log; public OrderService( IRepo repo, ILogger log) { _repo = repo; _log = log; } public void Process() => _repo.Save(); }
✓ C# 12 — Clean
// Parameters declared inline! public class OrderService( IRepo repo, ILogger log) { // Both in scope everywhere public void Process() => repo.Save(); public void Log(string m) => log.LogInfo(m); } // ~50% less code!

Spread Operator ..

The spread element .. inlines another collection inside a collection expression — like JavaScript spread but for all types.

Spread ..C# 12
int[] first = [1, 2, 3]; int[] second = [4, 5, 6]; // ✅ Merge two collections int[] all = [..first, ..second]; // → [1, 2, 3, 4, 5, 6] // ✅ Mix literals and spreads int[] mixed = [0, ..first, 99, ..second]; // → [0, 1, 2, 3, 99, 4, 5, 6] // ✅ Real world: combine validation errors List<string> allErrors = [..ValidateName(n), ..ValidateEmail(e)]; // ✅ Route building string[] baseRoute = ["api", "v1"]; string[] route = [..baseRoute, "orders", "details"]; // → ["api","v1","orders","details"]

✦ New in C# 13
params Now Works with Any Collection Type

C# 13 removes the restriction that params only supports arrays. Now any collection type works — including ReadOnlySpan<T> for zero-allocation params.

✗ C# 12 — Array only
// Heap alloc every call! void Print(params int[] nums) { } Print(1, 2, 3); // ⚠️ array alloc
✓ C# 13 — Any type
// Zero allocation! void Print(params ReadOnlySpan<int> n) { } Print(1, 2, 3); // ✅ stack only
params with various collection typesC# 13
// ✅ Span — zero allocation (best for hot paths) public int Sum(params ReadOnlySpan<int> nums) { int total = 0; foreach (var n in nums) total += n; return total; } // ✅ IEnumerable — most flexible public void AddAll(params IEnumerable<string> items) => _list.AddRange(items); // ✅ List<T> — mutable collection public void Process(params List<Order> orders) { } // ✅ Real world: high-perf logger public void LogError( string template, params ReadOnlySpan<object?> args) { // No array allocated when calling: // logger.LogError("{Name} failed", userName, orderId) Console.Error.WriteLine( string.Format(template, args.ToArray())); } Sum(1, 2, 3, 4, 5); // ✅ No heap alloc!

Overload Priority & Real Use Cases

System.Threading.Lock is a new purpose-built locking type — better diagnostics, performance, and ergonomics than lock(object).

✗ Before — lock(object)
private readonly object _lk = new(); public void Update() { lock (_lk) { // works but object overhead _value++; } }
✓ C# 13 — Lock type
private readonly Lock _lk = new(); public void Update() { lock (_lk) { // same syntax, better runtime _value++; } }
Lock — full feature setC# 13
using System.Threading; public class SafeCounter { private readonly Lock _lock = new(); private int _count = 0; // ✅ lock keyword — compiles to Lock.EnterScope public void Increment() { lock (_lock) { _count++; } } // ✅ Scope pattern — RAII, exception safe public int GetAndReset() { using (Lock.Scope scope = _lock.EnterScope()) { int val = _count; _count = 0; return val; } // auto-released here } // ✅ TryEnter — non-blocking attempt public bool TryUpdate() { if (_lock.TryEnter(out Lock.Scope scope)) { using (scope) { _count++; return true; } } return false; // another thread holds it } // ✅ IsHeldByCurrentThread public bool IsLocked => _lock.IsHeldByCurrentThread; }
Thread-Safe Repository Example
Production thread-safe storeC# 13
public class InMemoryUserStore { private readonly Lock _lock = new(); private readonly Dictionary<int, User> _users = new(); public User? Get(int id) { lock (_lock) return _users.TryGetValue(id, out var u) ? u : null; } // Non-blocking add — skip if locked public bool TryAdd(User user) { if (!_lock.TryEnter(TimeSpan.FromMilliseconds(50))) return false; try { return _users.TryAdd(user.Id, user); } finally { _lock.Exit(); } } // Atomic read-and-update public void UpdateLastSeen(int id) { using (_lock.EnterScope()) { if (_users.TryGetValue(id, out var u)) u.LastSeen = DateTime.UtcNow; } } }

Task.WhenEach returns an IAsyncEnumerable yielding tasks as they complete — don't wait for the slowest!

✗ WhenAll — waits for slowest

await foreach (var t in Task.WhenEach(tasks)) { var r = await t; Process(r); // ✅ Immediate! }

// ✅ Stream search results as providers respond public async IAsyncEnumerable<SearchResult> SearchAllAsync(string query) { var tasks = new[] { SearchGoogleAsync(query), SearchBingAsync(query), SearchYandexAsync(query) }; // Yield results AS EACH provider responds await foreach (var task in Task.WhenEach(tasks)) { SearchResult result = await task; yield return result; // → UI immediately! } // Google responds first → user sees it first! } // ✅ Progress reporting for parallel downloads await foreach (var done in Task.WhenEach(downloads)) { var file = await done; progressBar.Increment(); statusLabel.Text = $"Downloaded: {file.Name}"; }

The field keyword inside a property accessor refers to the compiler-generated backing field — eliminating the need to manually declare a backing field just to add setter validation or getter transformation.

This was GitHub's #1 most voted C# feature request for years.

✓ C# 14 — field keyword

C# 12 & .NET 8
C# 12 / .NET 8 — Questions & Answers
What are primary constructors and how do they differ from record primary constructors?
C# 12 primary constructors declare params in the class declaration and are in scope throughout the class. Unlike records, parameters are NOT automatically stored as fields or exposed as properties — you explicitly capture what you need. Records auto-generate init properties.
What is the spread element (..) in collection expressions?
The spread operator .. inlines another collection: [..list1, ..list2, extra]. Works with any IEnumerable. Compiler picks efficient allocation — for Span, no heap allocation occurs.
When would you NOT use primary constructors?
Avoid when you need multiple constructor overloads, complex initialization requiring try/catch, or when constructor logic needs significant transformation before storing. Regular constructors give more control in those cases.
What is Native AOT and what are its main limitations?
AOT compiles to native code at build time — no JIT, no .NET runtime needed. Benefits: ~5ms startup, ~20MB RAM, single binary. Limitations: no runtime reflection, no dynamic code gen, limited dynamic loading. Requires source generators for JSON/DI.
What is FrozenDictionary and how does it differ from ImmutableDictionary?
FrozenDictionary spends extra time at creation optimizing for max read speed. ImmutableDictionary supports functional updates but is O(log n). FrozenDictionary: fastest reads, truly immutable. ImmutableDictionary: for persistent data with updates.
What problem does TimeProvider solve in unit testing?
DateTime.Now is uncontrollable in tests. TimeProvider injects a FakeTimeProvider in tests that lets you set/advance time precisely — test expiry, scheduled jobs, retry delays etc. Use TimeProvider.System in production.
What are Keyed Services and when to use them?
Register multiple implementations of same interface with different keys: AddKeyedSingleton<IStorage, AzureBlob>("azure"). Use for strategy patterns: storage backends, payment providers, notification channels. Resolve with [FromKeyedServices("key")].
How does Blazor United work in .NET 8?
.NET 8 unified Blazor Server + WASM into one app model with per-component render modes: InteractiveServer (SignalR), InteractiveWebAssembly, InteractiveAuto (server first, then WASM), static SSR. One codebase, mix rendering strategies.
What is SearchValues<T> and why is it faster?
SearchValues.Create("abc") pre-builds an optimized SIMD search structure for a fixed char/byte set. At creation the runtime picks the best algorithm (vectorized, lookup table, etc.) — much faster than repeated manual IndexOf in hot paths.
C# 13 & .NET 9
C# 13 / .NET 9 — Questions & Answers
Why is params ReadOnlySpan<T> better than params T[]?
Array params always allocate on the heap. params ReadOnlySpan<T> uses stack allocation — zero heap allocation for inline calls. In hot paths like logging and formatting called millions of times, this dramatically reduces GC pressure and improves throughput.
What improvements does System.Threading.Lock bring over lock(object)?
Lock.Scope for RAII/exception-safe locking, TryEnter with timeout, IsHeldByCurrentThread for diagnostics, better debugger visualization, improved non-contended performance by avoiding object monitor overhead. Drop-in upgrade.
What are CountBy and AggregateBy and how do they improve on GroupBy?
CountBy counts by key in one pass without materializing groups. AggregateBy applies accumulator per key. Both are single-pass, lazy, and more memory efficient than GroupBy().Select() which creates intermediate group objects and materializes collections.
What is Task.WhenEach and how does it differ from WhenAll?
WhenAll waits for all tasks; slowest blocks everything. WhenEach returns IAsyncEnumerable<Task<T>> yielding tasks as they complete. Process first results immediately — perfect for parallel search, download progress reporting, streaming UI updates.
What is HybridCache and how does it solve stampede?
HybridCache combines L1 (memory) + L2 (distributed/Redis). Stampede: 10,000 concurrent cache misses → only ONE factory call fires, others wait. Replaces IDistributedCache + IMemoryCache boilerplate with one GetOrCreateAsync call.
Why use Guid.CreateVersion7() over Guid.NewGuid() for DB keys?
UUID v7 is time-ordered — newer GUIDs are always greater. Random UUIDs (v4) insert anywhere in B-tree indexes causing page splits and fragmentation. UUID v7 inserts sequentially, reducing splits by up to 50% on SQL Server/PostgreSQL clustered indexes.
C# 14 Preview & Future
C# 14 / Future — Questions & Answers
What is the field keyword and why was it so requested?
field refers to the compiler-generated backing field inside a property. Eliminates declaring a separate backing field just to add setter validation or getter transformation. Was GitHub's #1 C# feature request for years — addresses extremely common property validation patterns.
How are extension members different from extension methods?
Extension methods only add methods. C# 14 extension members use extension(...) { } blocks to add properties, operators, and static members too. Extension properties let you add computed read-only properties to types you don't own — powerful for domain enrichment.
What is .NET 10 and when should I upgrade?
.NET 10 is the next LTS (Nov 2025, 3-year support). The recommended upgrade path is .NET 8 LTS → .NET 10 LTS, skipping .NET 9 STS. For production, always favor LTS: .NET 6, 8, 10, 12 etc. STS for apps that can upgrade frequently.
What are Discriminated Unions and why does C# need them?
DUs represent a value that is exactly one of several types: union Result = Ok | Error. C# currently simulates this with sealed class hierarchies or libraries like OneOf. Native DUs would enable exhaustive compile-time pattern matching and eliminate null-or-value anti-patterns.
What LTS strategy should teams follow for .NET versions?
Use LTS releases (even-numbered) for production. LTS = 3 years support, STS = 18 months. Upgrade every LTS cycle. Never let a version fall out of support. Microsoft's guidance: be on a supported version within 6 months of release.
What are the most impactful features across all three versions combined?
Top picks: (1) Primary constructors + collection expressions [C# 12 — daily use], (2) params Span + HybridCache [C# 13/.NET 9 — performance], (3) field keyword + extension members [C# 14 — clean code], (4) Native AOT + Frozen collections [.NET 8 — architecture].
