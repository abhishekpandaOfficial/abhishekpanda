Exact Commands — Merge Phase 1 + Phase 2

Run these commands in order. This creates the full project from both zip files.

01

Unzip both phase files into one folder

bash
# Create your main project folder
mkdir openowl
cd openowl

# Unzip Phase 1 (the base)
unzip ~/Downloads/openowl-phase1.zip -d .
# This creates: openowl/ folder with all Phase 1 files inside

# Unzip Phase 2 into a temp folder
unzip ~/Downloads/openowl-phase2.zip -d /tmp/phase2
02

Copy Phase 2 files into the project

bash · run from inside your openowl/ folder
# Go into the project folder
cd openowl

# Copy Phase 2 new files/folders in
cp /tmp/phase2/openowl-phase2/channels/whatsapp_handler.py  channels/
cp /tmp/phase2/openowl-phase2/tools/google_calendar.py      tools/
cp /tmp/phase2/openowl-phase2/tools/gmail.py                tools/
cp /tmp/phase2/openowl-phase2/tools/web_tools.py            tools/
cp /tmp/phase2/openowl-phase2/integrations/google_oauth.py  integrations/
cp /tmp/phase2/openowl-phase2/scheduler/proactive.py        scheduler/

# Replace main.py with Phase 2 version (it includes Phase 1 features + new)
cp /tmp/phase2/openowl-phase2/main.py  main.py
03

Create missing __init__.py files

bash
# Create all __init__.py files so Python treats folders as packages
touch agent/__init__.py
touch channels/__init__.py
touch memory/__init__.py
touch models/__init__.py
touch personas/__init__.py
touch tools/__init__.py
touch integrations/__init__.py
touch scheduler/__init__.py
touch scripts/__init__.py
touch tests/__init__.py

# Create data folder (for Google OAuth tokens)
mkdir -p data
touch data/.gitkeep
04

Create .gitignore so you don't commit secrets

bash
cat > .gitignore << 'EOF'
.env
data/
__pycache__/
*.pyc
*.pyo
.pytest_cache/
.venv/
venv/
*.egg-info/
.DS_Store
Thumbs.db
*.log
EOF
⚠️ Critical — never commit .env

Your .env file contains API keys. The .gitignore above prevents it from being pushed to GitHub. Always check before committing.

05

Merge requirements into one file

bash
# Append Phase 2 packages to main requirements.txt
cat >> requirements.txt << 'EOF'

# ── Phase 2 additions ──────────────────────────────────

06

Create Python virtual environment and install

bash
# Create virtual environment (named .venv)
python -m venv .venv

# Activate it
source .venv/bin/activate           # Mac / Linux / WSL
# .venv\Scripts\activate           # Windows PowerShell

# Install all packages
pip install --upgrade pip
pip install -r requirements.txt

# Verify key packages installed
python -c "import fastapi, langgraph, groq; print('✅ Core packages OK')"
✅ After these 6 steps

Your folder is complete. Run ls -la inside openowl/ — you should see: main.py config.py agent/ channels/ memory/ models/ personas/ tools/ integrations/ scheduler/ data/ .env.example .gitignore requirements.txt Dockerfile docker-compose.yml

  "python.linting.pylintEnabled": false,

  "editor.rulers": [88],

cat > .vscode/launch.json << 'EOF'
{

  "configurations": [
    {
      "name": "🦉 Run OpenOwl",

      "request": "launch",
      "module": "uvicorn",

Now pressing F5 in VS Code starts the server with hot-reload.

Create Your Telegram Bot — Step by Step

1

Open Telegram and find @BotFather

On your phone or Telegram desktop app, search for @BotFather — it's the official bot for creating bots. The blue tick ✅ confirms it's official.

2

Create a new bot

telegram conversation with @BotFather
You:   /newbot

BotFather: Alright, a new bot. How are we going to call it?
           Please choose a name for your bot.

You:   OpenOwl Assistant
       (this is the display name, can be anything)

BotFather: Good. Now let's choose a username for your bot.
           It must end in 'bot'. Like this, for example:
           TetrisBot or tetris_bot.

You:   MyOpenOwlBot
       (must be unique + end in Bot or bot)

BotFather: Done! Congratulations on your new bot.
           Use this token to access the HTTP API:
           7234567890:AAEb3KmXgH9xxxxxxxxxxxxxxxZ4Qabcd
                      ↑ THIS IS YOUR BOT TOKEN — copy it
Save that token immediately

Copy the token BotFather gives you. It looks like 7234567890:AAEb3KmXgH9xZ4Qabcd. You'll paste it into .env as TELEGRAM_BOT_TOKEN. Don't share this — it gives full control of your bot.

3

Get your own Telegram user ID

telegram · find @userinfobot
Search: @userinfobot

You:   /start

userinfobot: Id: 123456789
             First: YourName
                   ↑ This is your OWNER_ID — copy it

You need this so OpenOwl knows you're the owner and trusts your commands.

4

Configure bot commands (optional but nice)

telegram · back to @BotFather
You: /setcommands

BotFather: Choose a bot to change the list of commands.
You: @MyOpenOwlBot

BotFather: OK. Send me a list of commands...
You:
start - Welcome and setup
help - Show all commands
switch - Change persona (aria/priya/nova/meera/zara)
status - System health check
memory - See what Owl remembers

Connect the Bot — Get Groq API Key + Configure

1

Get your Groq API key (free — takes 2 minutes)

Groq is the free AI API that powers OpenOwl. No credit card needed.

steps
1. Go to: console.groq.com
2. Click "Sign Up" → use Google account
3. Go to API Keys → Create API Key
4. Copy the key → looks like: gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
5. Paste into .env as GROQ_API_KEY
2

Create your .env file from the template

bash · inside openowl/ folder
cp .env.example .env

Now open .env in VS Code and fill in just these 3 fields to start:

.env · minimum to get running
# The ONLY 3 things you MUST fill in to start

TELEGRAM_BOT_TOKEN=7234567890:AAEb3KmXgH9xZ4Qabcd
                   ↑ paste your BotFather token here

GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
             ↑ paste your Groq key here

OWNER_ID=123456789
         ↑ paste your Telegram user ID here (from @userinfobot)

# TELEGRAM_WEBHOOK_URL gets filled in the NEXT step (after ngrok)
# Everything else can stay as default for now

ngrok — Make Your Local Server Reachable by Telegram

Telegram needs a public HTTPS URL to send messages to your bot. ngrok creates a tunnel from the internet to your local machine.

1

Install and authenticate ngrok

bash
# Mac
brew install ngrok

# Linux / WSL
curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc >/dev/null
echo "deb https://ngrok-agent.s3.amazonaws.com buster main" | sudo tee /etc/apt/sources.list.d/ngrok.list
sudo apt update && sudo apt install ngrok

# Authenticate (get token from dashboard.ngrok.com → Your Authtoken)
ngrok config add-authtoken YOUR_NGROK_TOKEN
2

Start Docker services (Redis + PostgreSQL)

bash · terminal window 1
# Make sure Docker Desktop is running, then:
docker-compose up -d postgres redis

# Check they're running
docker ps
# Should show: openowl-postgres and openowl-redis both "Up"
3

Start the OpenOwl server

bash · terminal window 2 (or press F5 in VS Code)
# Make sure .venv is active
source .venv/bin/activate

# Start the server
uvicorn main:app --reload --port 8000

# You should see:
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     ✅ Redis connected
INFO:     ✅ PostgreSQL connected
INFO:     ✅ OpenOwl is ready!
4

Start ngrok in a new terminal

bash · terminal window 3
ngrok http 8000

# ngrok shows something like this:
Forwarding  https://a1b2c3d4.ngrok-free.app → http://localhost:8000
                 ↑ COPY THIS URL
5

Paste the ngrok URL into .env and restart

.env · add the ngrok URL
TELEGRAM_WEBHOOK_URL=https://a1b2c3d4.ngrok-free.app
                      ↑ paste your ngrok URL here (no trailing slash)
bash · restart the server to pick up the new URL
# Stop the server (Ctrl+C) then restart
uvicorn main:app --reload --port 8000

# You should now see:
INFO:     ✅ Telegram webhook set → @MyOpenOwlBot
INFO:     Webhook URL: https://a1b2c3d4.ngrok-free.app/telegram/webhook
💡 ngrok free tier resets URL on restart

Every time you restart ngrok, you get a new URL. You'll need to update TELEGRAM_WEBHOOK_URL in .env and restart the server each time. For a permanent URL, upgrade ngrok to paid ($10/mo) or deploy to Railway (free tier).

Send Your First Message — Test It Works

With everything running, open Telegram and find your bot by its username (e.g. @MyOpenOwlBot). Send these in order:

telegram · test sequence
Test 1 — Basic response
You:  /start
Owl:  🦉 Welcome to OpenOwl! I'm Aria, your PA...

Test 2 — Free text conversation
You:  Hello! What can you do?
Owl:  Hey! I'm Aria, your personal assistant. I can book flights,
      manage your calendar, set reminders, research anything...

Test 3 — AI working (Groq)
You:  Explain quantum computing in simple words
Owl:  [Groq model responds — should be <2 seconds]

Test 4 — Persona switch
You:  Switch to Meera
Owl:  Namaste! Main Meera hoon 😊 Aaj main aapki kaise help...

Test 5 — Approval gate (safety test)
You:  Send an email to test@test.com saying hello
Owl:  Here's what I'm about to do — confirm to proceed:
      Action: Send email to test@test.com
      [YES button] [NO button]

Test 6 — System status
You:  /status
Owl:  🦉 OpenOwl Status
      AI Model: 🟢 Groq (OSS)
      Redis: 🟢 Connected
      Database: 🟢 Connected
✅ If all 6 tests pass — you're live!

Your OpenOwl is fully working. The bot is connected, the AI is responding, memory is storing, and the approval gate is active. You can now start adding integrations (Google Calendar, Gmail) and testing more complex tasks.

❌ If the bot doesn't respond

Check in order: (1) Is the server running? Check terminal 2. (2) Is ngrok running? Check terminal 3. (3) Is TELEGRAM_WEBHOOK_URL set in .env to the current ngrok URL? (4) Did you restart the server after changing .env? (5) Visit http://localhost:8000/health — does it say "ok"?

📱 Test on your phone too

Open Telegram on your phone, find your bot — it works on mobile too. That's the whole point: you can message it from anywhere and it responds instantly.
